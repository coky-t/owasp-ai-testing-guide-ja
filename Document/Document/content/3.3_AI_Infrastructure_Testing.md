
# 3.3 AI インフラストラクチャテスト (AI Infrastructure Testing)

**AI インフラストラクチャテスト** カテゴリは AI モデルのデプロイメントとオペレーションをサポートする技術インフラストラクチャとコンポーネント内の脆弱性とリスクをターゲットとしています。このカテゴリは、モデルサプライチェーン、リソース管理、境界制御、プラグイン、ファインチューニング環境、不正なモデルアクセスや不正使用を防ぐためのメカニズムなど、インフラストラクチャレベルのセキュリティを具体的に検査します。

インフラストラクチャレベルの脆弱性は、不正アクセス、リソース枯渇、モデルや関連コンポーネントの改竄といった重大な問題につながる可能性があります。包括的なインフラストラクチャテストは、システムが安全に構成され、不正使用や悪用に対する耐性があり、サポートする AI システムを保護する能力を確保します。

### このテストカテゴリのスコープ (Scope of This Testing Category)

このカテゴリは AI インフラストラクチャが以下であるかどうかを評価します。

- **サプライチェーン改竄と不正な改変** を防止する  
  → [AITG-INF-01: サプライチェーン改竄のテスト (Testing for Supply Chain Tampering)](tests/AITG-INF-01_Testing_for_Supply_Chain_Tampering.md)

- **リソース枯渇とサービス拒否状態** に対して耐性がある  
  → [AITG-INF-02: リソース枯渇のテスト (Testing for Resource Exhaustion)](tests/AITG-INF-02_Testing_for_Resource_Exhaustion.md)

- **プラグインベースのインタラクションのための** 安全な **境界とアクセス制御** を維持する  
  → [AITG-INF-03: プラグイン境界違反のテスト (Testing for Plugin Boundary Violations)](tests/AITG-INF-03_Testing_for_Plugin_Boundary_Violations.md)

- **モデルのケイパビリティと機能の不正使用** に対して厳格な制御を実施する  
  → [AITG-INF-04: ケイパビリティ不正使用のテスト (Testing for Capability Misuse)](tests/AITG-INF-04_Testing_for_Capability_Misuse.md)

- **ポイズニングや破損に対してモデルのファインチューニング** に使用される環境を保護する  
  → [AITG-INF-05: ファインチューニングポイズニングのテスト (Testing for Fine-tuning Poisoning)](tests/AITG-INF-05_Testing_for_Fine-tuning_Poisoning.md)

- **開発フェーズでのモデルの窃取や漏洩** を防止する  
  → [AITG-INF-06: 開発時のモデル窃取のテスト (Testing for Dev-Time Model Theft)](tests/AITG-INF-06_Testing_for_Dev-Time_Model_Theft.md)

AI インフラストラクチャテストカテゴリ内の各テストは AI システムに必要な基本的なセキュリティ態勢に貢献し、モデルのライフサイクル全体にわたって脅威を防止および緩和できる、信頼性が高く、安全で、堅牢なインフラストラクチャを確保します。
